# PARITY REPORT — omegacode vs Claude Code Workflows

## Verdict

**omegacode is a faithful re-implementation of Claude Code's workflow *execution engine* (vm sandbox, chained-key journal, longest-prefix replay, `parallel`/`pipeline` semantics are at parity), but it is missing the entire model-facing tool layer, the named-workflow registry, the interactive `/workflows` control UI, the `budget`/`workflow()` globals, and several sandbox-hardening/determinism seals.**

**Estimated parity: ~60% overall** — ~75% on the core authoring/runtime engine, ~35% on the Workflow-tool + UI + registry surface.

Tally across 98 catalogued features: **26 parity · 36 partial · 23 missing · 13 extra**.

---

## Gaps

### 🔴 Blockers (break compatibility, correctness, or safety)

**Authoring-surface globals that A-authored scripts reference (undefined-global crash in B):**
- **`budget` global** *(missing)* — Build a frozen `budget = { total, spent(), remaining() }` injected into the sandbox; gate `agent`/`parallel`/`pipeline` on it (total may be `null`/inert as in A, but the surface must exist). You already aggregate `totalUsage` via `addUsage` — wire it to `spent()`.
- **`workflow(nameOrRef, args?)` nested primitive** *(missing)* — Inject a one-level nested-workflow global sharing the run's concurrency cap, agent cap, abort signal, and journal; reject `workflow()` inside a child.
- **`agent({ agentType })`** *(missing)* — Add `agentType` to `AgentOpts`, resolve against a subagent registry, and include it in `keyedOpts` (it must invalidate the resume cache).

**Determinism / replay correctness (the journal's entire reason to exist):**
- **`new Date(x).constructor` backdoor + unfrozen `RealDate`/`Math`** *(missing)* — `ShimDate.prototype = RealDate.prototype` leaves `RealDate.prototype.constructor === RealDate`, so `(new Date(0)).constructor.now()` reaches the real clock; the static lint won't catch this form. Rewire `RealDate.prototype.constructor` and `Object.freeze(RealDate)`. **This is a reproducible, silent replay-soundness hole.**
- **Frozen intrinsics / SES-style hardening** *(missing)* — Freeze `Promise/Object/Array/Date/Math/...` and apply override guards so the determinism shims can't be reassigned off an unfrozen global.

**Meta safety:**
- **Strict pure-literal `meta` evaluator** *(missing)* — B does `new Script('(' + metaSrc + ')')`, which *executes* method calls, computed keys, spread, and reserved keys (`__proto__`/`constructor`) that A rejects at the AST level. Replace the regex+VM-eval with an acorn parse + literal-only evaluator anchored to the first statement.

**Tool / integration layer (entire class missing — B is CLI-first):**
- **Model-facing `Workflow` tool, inline `{script}` input, `name` input, async-launch result (`taskId`/`runId`/task-notification)** *(all missing)* — Required for tool-call parity. Build a tool entrypoint that accepts `{script | scriptPath | name, args, resumeFromRunId}`, persists inline scripts to `scripts/<slug>-<runId>.js`, launches detached, and returns immediately.
- **Named-workflow registry** *(missing)* — Add user/project `.claude/workflows` scanning (project shadows user) + built-ins, so `name` resolves and `validate`/`run` accept names, not just file paths.
- **Workflow-level approval gate** *(missing)* — Add a pre-launch `checkPermissions`/confirmation step (B defaults agent approval to `never` and runs immediately — no gate at all).

### 🟡 Nice-to-have (functional in the common case, weaker than A)

- **`meta.phases` pre-seeding + exact-title progress matching** *(missing)* — pass `meta.phases` into the `Runtime` and pre-seed groups instead of creating them lazily in `phase()`.
- **`agent({ phase })` override** *(partial)* — typed but ignored; `resolveSpec` must honor it instead of always using `this.currentPhase`.
- **`stallMs` watchdog + 5× stalled-agent retry** *(missing)* — add a stall watchdog disarmed during pending tool calls.
- **User skip/retry → `agent()` resolves `null`** *(missing)* — no skip path exists; the `skipped` event state is unused.
- **StructuredOutput corrective retry** *(partial)* — DESIGN promises "one corrective retry then fail," but `finalizeResult` throws immediately on a validation miss. Implement the single nudge/retry before throwing.
- **`diverged` latch** *(missing)* and **started-hit-respawn detection/telemetry** *(partial — `loaded.startedOnly` is dead code)*.
- **Resume guard for still-running run** *(missing)* — add a liveness/lock check so two processes can't append to one `journal.jsonl`.
- **`--resume-last` / `latestRunIdForFile`** *(stub)*, and **resume falling back to journaled `args`** *(partial)* — `run.ts:91` uses `opts.args` directly.
- **`pipeline` stages-array fan-out cap + VM-boundary cap on return values/args** *(partial)* — only input arrays are length-checked.
- **`console` routed through `log()` + final-log capture cap** *(partial)* — currently injected as host console.
- **VM-boundary `structuredClone`/JSON-serializability assertion on return** *(partial)*.
- **Host-error isolation at the VM boundary** *(missing)*; **dangerous globals deletion** (`WeakRef`, `Atomics`, `SharedArrayBuffer`, `queueMicrotask`, …) *(partial)*; **await trampoline** *(missing)* — sandbox hardening.
- **`meta.title` + `meta.phases[].model` + optional-field shape validation** *(partial/missing)*.
- **Live progress tree** *(partial — line-stream only)*, **web-viewer full drill-down/expandable prompts** *(partial)*, **UI stop/restart/pause/save controls** *(missing)*, **saving a run as a named workflow** *(missing)*, **surfacing built-in deep-research/code-review as commands** *(partial — examples only)*.
- **Codex structured-output extraction** *(partial)* — best-effort JSON-parse of assistant text is fragile vs a dedicated channel.
- **`maxTurns` not in `keyedOpts`** — changing it silently doesn't invalidate the cache, and Codex ignores it; either key it or document the limitation.

## Extras (omegacode exceeds Claude Code)

- **Seeded deterministic `now()` / `random()` globals** — journal-persisted seed + `baseTimeMs` reproduce exactly on replay; gives authors *legitimate* time/RNG where A simply bans them.
- **Explicit `agent({ key })` resume pin** — survives reordering/prompt-wording edits; A's chained key cannot be pinned.
- **Native provider structured output** — `outputFormat`/`outputSchema` instead of an injected `StructuredOutput` tool.
- **Provider-neutral controls** — `provider`, `effort`, `cwd`, `sandbox`, `approval`, `instructions`, `maxTurns`, `worktree`, plus `meta.defaultProvider/defaultModel/defaultSandbox` and worker-factory provider resolution.
- **Configurable `--concurrency`** (A's cap is fixed/CPU-derived, not user-settable).
- **Cross-session resume** — on-disk `~/.omegacode` journal (not session-scoped) survives separate CLI invocations; A is same-session only.
- **Richer journal records** — `JournalMeta` (runId/fileHash/args/seed/createdAt) + per-result `usage`/`provider`/`worktreeBranch`/`durationMs`.
- **Standalone localhost HTTP viewer** with `/api/runs` + SSE stream (B's substitute for `/workflows`).
- **Per-agent approval/sandbox mapping** (read-only `canUseTool` deny, Codex `approvalPolicy`).
- **Lint runs over whole source, always** (B is more consistent than A's inline-only, meta-stripped lint).

## Prioritized punch-list to 100% parity

1. **Seal the determinism holes** (correctness): rewire `RealDate.prototype.constructor`, `Object.freeze` `RealDate`/`Math`, freeze intrinsics. *(Replay soundness — highest risk.)*
2. **Replace the meta evaluator** with an acorn pure-literal parser anchored to the first statement; reject code in `meta`. *(Security/correctness.)*
3. **Inject `budget` and `workflow()` globals + add `agentType`** — close the three authoring-incompatible global/opt gaps so A-authored scripts run unchanged. Key `agentType`.
4. **Build the model-facing `Workflow` tool**: `{script | scriptPath | name}` inputs, inline-script persistence, async-launch result (`taskId`/`runId`/notification).
5. **Add the named-workflow registry** (user/project/built-in tiers) so `name` resolves and built-ins surface as commands.
6. **Add the workflow-level approval gate** + a resume still-running guard. *(Safety.)*
7. **Implement the promised StructuredOutput corrective retry**, the `diverged` latch, and started-hit-respawn detection/telemetry.
8. **Honor `agent({ phase })`**, pre-seed `meta.phases`, add `stallMs`/skip-null, full `meta` field validation, and the missing fan-out/boundary caps.
9. **Upgrade observability**: live progress tree, viewer drill-down, and stop/restart/pause/save controls (or formally scope these as the intentional CLI-first divergence).

Items 1–3 are the true correctness/compatibility blockers; 4–6 are the integration surface that most of the "missing" count comes from; 7–9 close robustness and UX parity.
